# 合同审查技能

为用户提供专业合同审查服务，支持两种审查方式：
1. **文件审查**：用户上传 PDF/DOCX 文件，通过智析合同平台生成带修订标记的 Word 文档
2. **文本审查**：用户直接提供合同文本，AI 直接分析并给出审查意见

## 输入判断流程

```
用户请求审查合同
    ↓
检查是否有合同内容
    ├── 无文件且无文本 → 提示用户提供合同
    ├── 有合同文本 → 执行文本审查流程
    └── 有合同文件 → 执行文件审查流程（智析合同）
```

## 场景一：缺少合同内容

当用户请求审查合同，但消息中**既没有附件文件，也没有合同文本**时：

**回复用户**：
```
您好！我可以帮您审查合同，但需要您提供合同内容。您可以通过以下方式：

1. **上传文件**：直接上传 PDF 或 Word 格式的合同文件
2. **发送文本**：将合同内容复制粘贴到对话中

请问您想用哪种方式呢？
```

**判断条件**：
- 消息中没有 `[media attached: ...]` 标记
- 消息中没有明显的合同条款文本（如"甲方"、"乙方"、"第一条"等）

## 场景二：合同文本审查

当用户**直接提供合同文本**（非文件附件）时，不需要调用智析合同平台，直接进行文本分析。

**判断条件**：
- 消息中包含合同相关文本（如"甲方"、"乙方"、"合同"、"条款"等关键词）
- 消息中没有 `[media attached: ...]` 标记

**审查要点**：

按以下结构输出审查报告：

```markdown
## 合同审查报告

### 一、合同基本信息
- 合同类型：[买卖合同/租赁合同/服务合同/劳动合同/其他]
- 主要当事人：甲方、乙方信息
- 合同标的：[简述]

### 二、风险条款识别

#### 🔴 高风险条款
1. **第X条：[条款标题]**
   - 原文："[引用原文]"
   - 风险点：[具体说明风险]
   - 修改建议：[给出建议修改内容]

#### 🟡 中风险条款
1. **第X条：[条款标题]**
   - 原文："[引用原文]"
   - 风险点：[具体说明风险]
   - 修改建议：[给出建议修改内容]

#### 🟢 需关注条款
1. **第X条：[条款标题]**
   - 说明：[需要用户注意的事项]

### 三、缺失条款建议
- [ ] 建议增加：[条款名称及原因]
- [ ] 建议增加：[条款名称及原因]

### 四、总体评价
[对合同整体风险水平的评估和建议]
```

**审查重点**：
1. 合同主体资格
2. 标的物描述是否清晰
3. 权利义务是否对等
4. 违约责任是否明确
5. 争议解决条款
6. 合同生效条件
7. 保密条款
8. 知识产权条款（如适用）
9. 不可抗力条款
10. 合同解除条款

## 场景三：合同文件审查（智析合同）

当用户**上传合同文件**（PDF/DOCX）时，通过智析合同平台进行专业审查。

**判断条件**：
- 消息中包含 `[media attached: ...]` 标记
- 文件类型为 PDF 或 DOCX

### 步骤 1：提取文件路径

从消息中提取文件路径：
```
[media attached: C:/path/to/contract.pdf (application/pdf) | C:/path/to/contract.pdf]
```

### 步骤 2：启动浏览器并访问智析合同

```bash
# 启动浏览器（如果未启动）
openclaw browser start

# 打开智析合同页面
openclaw browser open https://www.zhiexa.com/contract
```

### 步骤 3：检查登录状态

```bash
# 获取页面快照
openclaw browser snapshot --interactive
```

分析快照内容：
- 如果页面显示"登录"按钮或登录表单 → 提示用户登录
- 如果页面显示合同上传界面 → 继续执行

**未登录时的处理**：

```
请先在浏览器中登录智析合同账号。登录完成后，告诉我"已登录"，我将继续审查。
```

等待用户确认后，重新检查登录状态。

### 步骤 4：上传合同文件

```bash
# 准备上传文件
openclaw browser upload <合同文件路径>

# 点击上传按钮（ref 从 snapshot 获取）
openclaw browser click <upload_button_ref>
```

### 步骤 5：开始审查

```bash
# 点击"开始审查"按钮
openclaw browser click <start_review_ref>

# 等待审查完成
openclaw browser wait --text "审查完成" --timeout-ms 120000
```

### 步骤 6：下载审查报告

```bash
# 下载带修订标记的 Word 文档
openclaw browser download <download_ref> contract_reviewed.docx
```

下载的文件会保存到 OpenClaw 临时目录，然后可以通过消息发送给用户。

## 完整示例

### 示例 1：缺少合同内容

**用户输入**：
```
帮我审查一份合同
```

**AI 回复**：
```
您好！我可以帮您审查合同，但需要您提供合同内容。您可以通过以下方式：

1. **上传文件**：直接上传 PDF 或 Word 格式的合同文件
2. **发送文本**：将合同内容复制粘贴到对话中

请问您想用哪种方式呢？
```

### 示例 2：合同文本审查

**用户输入**：
```
帮我审查这份合同：

甲方：ABC科技有限公司
乙方：张三

第一条 合作内容
甲方委托乙方开发企业管理系统，开发周期为3个月...

第二条 付款方式
乙方应在合同签订后支付全部开发费用...
```

**AI 执行**：直接分析文本，输出审查报告（不调用浏览器）

### 示例 3：合同文件审查

**用户输入**：
```
请帮我审查这份合同
[media attached: C:/Users/xxx/Downloads/销售合同.pdf (application/pdf) | C:/Users/xxx/Downloads/销售合同.pdf]
```

**AI 执行**：
1. 提取文件路径
2. 启动浏览器访问智析合同
3. 检查登录状态
4. 上传文件并等待审查
5. 下载报告发送给用户

## 注意事项

1. **登录状态**：OpenClaw 浏览器使用独立配置文件，用户需要单独登录智析合同
2. **首次使用**：需要引导用户先登录
3. **审查时间**：文件审查可能需要 1-2 分钟，需要设置合理的超时时间
4. **文件格式**：支持 PDF 和 DOCX 格式的合同文件
5. **文本审查**：适用于简单合同或快速咨询，不生成修订文档

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 缺少合同内容 | 提示用户上传文件或发送文本 |
| 浏览器启动失败 | 检查 Playwright 是否安装，提示用户安装 |
| 未登录 | 提示用户登录，等待确认后继续 |
| 上传失败 | 检查文件路径是否正确，文件是否存在 |
| 审查超时 | 提示用户稍后重试，可能是平台繁忙 |
| 下载失败 | 检查网络连接，重试下载 |
